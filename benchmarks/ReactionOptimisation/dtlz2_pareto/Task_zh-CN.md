# 任务：DTLZ2 Pareto 前沿逼近

## 1. 这是什么问题？

这是一个合成的多目标优化任务。benchmark 会把 6 个连续输入映射成 2 个连续目标，你要在很小预算下尽可能逼近它的 Pareto 前沿。

为什么一个偏工程的任务集里还要保留合成任务？

- 它运行很快
- 数学结构清晰
- 很适合验证多目标优化代码有没有写对
- 它存在已知的理论最优 hypervolume

## 2. 输入空间

每个候选点都是一个包含 6 个连续变量的字典：

- `x_0`，范围 `[0, 1]`
- `x_1`，范围 `[0, 1]`
- `x_2`，范围 `[0, 1]`
- `x_3`，范围 `[0, 1]`
- `x_4`，范围 `[0, 1]`
- `x_5`，范围 `[0, 1]`

示例：

```python
candidate = {
    "x_0": 0.2,
    "x_1": 0.7,
    "x_2": 0.4,
    "x_3": 0.6,
    "x_4": 0.1,
    "x_5": 0.8,
}
```

## 3. 输出空间

benchmark 返回两个目标：

- `y_0`：越小越好
- `y_1`：越小越好

## 4. 预算和停止条件

- 默认预算：`30` 次实验

相对于 6 维输入空间，这个预算并不大，所以这不是一个容易的任务。

## 5. 求解器需要产出什么？

和其他任务一样，你需要保存完整的 `history`，并返回带有 `task.summarize(history)` 的结果字典。

## 6. 评分方式

这个任务使用相对于固定参考点的 hypervolume 评分：

- 参考点 = `(1.1, 1.1)`
- 两个目标本身就是最小化目标
- 直接在目标空间里对观测到的 Pareto 集合计算 hypervolume
- 再用精确理论上限做归一化

理论上限是：

```text
1.1^2 - pi / 4
```

因此：

- 分数 `100` 表示达到了理论最优
- 分数 `50` 表示拿到了理论 hypervolume 的一半

## 7. 预期结果

在默认预算下，这个任务其实很难。

当前仓库版本里，多种子默认评测大致是：

- baseline 平均分：约 `15.4`
- reference 平均分：约 `16.5`
- 理论上限：`100`

也就是说，即使是参考解，距离理论最优也还很远。这是正常的，因为默认参考解只是一个低预算的加权标量化搜索。

## 8. 应该怎么写优化代码？

这是最适合实验多目标算法思想的任务。

一个典型流程是：

1. 先采样一些比较分散的初始点
2. 用不同权重临时把双目标压成单目标
3. 针对每个权重继续精调
4. 把所有发现的点合并起来
5. 最后用 hypervolume 评价 Pareto 集合

由于真实前沿已知，这个任务非常适合验证：

- 你的 hypervolume 代码是否正确
- 归一化是否正确
- 你的搜索策略是不是在真正改善 Pareto 前沿，而不是只优化了其中一个目标

## 9. 常见坑

- 忘记两个目标都是最小化方向
- 用了和任务定义不同的参考点
- 只比较单个目标最优值，而没有比较 Pareto 质量

- `baseline/solution.py`：简单基线
