# 任务：Reizman Suzuki 仿真器 Pareto 优化

## 1. 这是什么问题？

这个任务模拟的是 Suzuki 偶联反应里的催化剂与工艺条件联合优化。你需要同时决定：

- 用哪一种催化剂
- 停留时间多长
- 温度设多少
- 催化剂投加量多大

这是一个多目标、预算受限的问题，所以很像真实实验 campaign 规划。

## 2. 输入空间

每个候选点都是一个字典，包含：

- `catalyst`，取值 `{P1-L1, P2-L1, P1-L2, P1-L3, P1-L4, P1-L5, P1-L6, P1-L7}`
- `t_res`，范围 `[60.0, 600.0]`
- `temperature`，范围 `[30.0, 110.0]`
- `catalyst_loading`，范围 `[0.5, 2.5]`

示例：

```python
candidate = {
    "catalyst": "P1-L4",
    "t_res": 280.0,
    "temperature": 105.0,
    "catalyst_loading": 2.1,
}
```

## 3. 输出空间

参与评分的输出有两个：

- `yld`：收率，越大越好
- `ton`：这个任务把它当作一种代价型 benchmark 输出，越小越好

一个非常重要的提醒：

在化学语境里，TON 这个缩写常见的含义和这里不一定一致。这个任务里不要猜测物理含义，直接以 `task.py` 中的定义为准：`yld` 越大越好，`ton` 越小越好。

## 4. 预算和停止条件

- 默认预算：`24` 次实验

你必须在极小预算里同时完成催化剂筛选和连续变量精调。

## 5. 求解器需要产出什么？

和其他任务一样，你的求解器需要维护一个 `history` 列表，保存所有观测记录，并返回带有 `summary` 的结果字典。

## 6. 评分方式

这是一个基于二维 hypervolume 的 Pareto 任务。

实现逻辑是：

- `yld` 用 `100` 归一化
- `ton` 先转成“越大越好”的形式：`1 - ton / 200`
- 两个值都截断到 `[0, 1]`
- 对观测到的折中解集合计算二维 hypervolume
- 最终得分 = `100 * hypervolume`

这意味着：

- 收率更高会加分
- `ton` 更低会加分
- 只优化一个方向不够，覆盖更多有效折中点会更好

## 7. 预期结果

在默认预算下：

- 纯随机算法有时会“撞大运”，但稳定性很差
- 更好的策略通常会先筛掉弱催化剂，再把预算集中到少数更有希望的催化剂和工艺区间

当前仓库版本里，多种子默认评测大致是：

- baseline 平均分：约 `63.5`
- reference 平均分：约 `81.8`

## 8. 应该怎么写优化代码？

一个很好的思考框架是：

- 第 1 阶段：低成本筛选催化剂类别
- 第 2 阶段：挑出几个有希望的催化剂 / 标量化权重组合
- 第 3 阶段：在这些子问题里继续优化 `t_res`、`temperature`、`catalyst_loading`

这很接近真实工程实践：

- 先排除明显差的选项
- 再把昂贵实验集中用在更值得投入的地方

你也可以在开发算法时用标量化技巧：

- 先选一个临时权重
- 把双目标临时压成单目标
- 连续优化若干步
- 再换几个权重重复，最后拼出 Pareto 解集

## 9. 常见坑

- 把大量预算都花在随机换催化剂上，会浪费数据。
- 只盯着最高收率的催化剂，可能得到很差的折中解。
- 如果忘了 `ton` 在这个任务里是“越小越好”，分数会算错。
